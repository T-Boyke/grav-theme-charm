# Mitwirken an diesem Projekt

Wir freuen uns über jeden Beitrag! Damit die Zusammenarbeit reibungslos klappt, beachte bitte folgende Richtlinien.

## Pull Requests (PR)
1.  **Forke** das Repository und erstelle einen Clone.
2.  Erstelle einen neuen Branch für dein Feature: `git checkout -b feature/mein-feature`.
3.  **Teste** deine Änderungen lokal.
4.  Achte auf sauberen Code und halte dich an die `.editorconfig`.
5.  Erstelle den Pull Request gegen den `main` Branch.

## Commit Messages
Wir folgen der [Conventional Commits](https://www.conventionalcommits.org/) Konvention:
* `feat: ...` für neue Features
* `fix: ...` für Bugfixes
* `docs: ...` für Änderungen an der Dokumentation
* `chore: ...` für Wartungsarbeiten (Updates, Configs)

## Fehler melden
Nutze bitte den "Issues"-Reiter in GitHub und beschreibe den Fehler so genau wie möglich (Schritte zum Reproduzieren, erwartetes vs. tatsächliches Verhalten).
